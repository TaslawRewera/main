# Подготовка 3D-моделей (мужская и женская)

## Выбор формата
- Основной формат: **GLB** (binary glTF).
- Единицы: метры.
- Ось: Y-up.

## Минимальные требования к обеим моделям
1. Отдельные меши по зонам клика (например: head, neck, chest, abdomen, pelvis, shoulder_l, shoulder_r и т.д.).
2. Уникальные `mesh_id` для сопоставления с `anatomy_zones` в БД.
3. Оптимизация: до 150k треугольников на модель для web-MVP.
4. 4 уровня LOD для плавного зума.
5. Текстуры с KTX2/Basis-сжатием.

## Пайплайн подготовки
1. Сбор исходников (анатомический референс/лицензия).
2. Ретопология и разметка зон в Blender.
3. Экспорт GLB.
4. Прогон через `gltf-transform optimize`.
5. Проверка в сцене Next.js (клик-зоны, производительность, корректность названий).

## Нейминг
- `male_body_v1.glb`
- `female_body_v1.glb`
- `zone_map_male_v1.json`
- `zone_map_female_v1.json`

## Definition of Done
- Модель вращается/зумится без лагов на среднем мобильном устройстве.
- Клик по каждой зоне возвращает корректный `zone_id`.
- Нет пересечений зон, которые ломают hit test.
