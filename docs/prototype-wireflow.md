# Кликабельный прототип: структура экранов (Figma + быстрый frontend)

3. `03_zone_zoom`
4. `04_pain_character`
5. `05_pain_intensity`
6. `06_additional_symptoms`
7. `07_result_conditions`
8. `08_result_specialists_redflags`

## User flow
- Пользователь выбирает пол → открывается 3D-модель.
- Пользователь отмечает зону боли → автоматический zoom-in.
- Пользователь отвечает на 2–3 вопроса о боли.
- Появляется ранжированный результат + рекомендованные специалисты.

## Быстрый frontend прототип (1 спринт)
- Страницы:
  - `/` — выбор пола.
  - `/viewer` — 3D-модель + выбор зоны.
  - `/questionnaire` — вопросы о боли.
  - `/result` — гипотезы и специалисты.
- Состояние: хранить шаги в Zustand-store.
- API пока мокать через `msw`.

## Acceptance criteria
- Не более 5 кликов до получения результата.
- В каждом шаге понятный CTA и индикатор прогресса.
- Отдельный блок "Когда срочно к врачу" присутствует на экране результата.
